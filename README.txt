================================================================================
                    Netlify CLI AI - 智慧部署助手
                       曾慶良(阿亮老師)建置
================================================================================

這是一個 Netlify + Python CLI AI 工具，幫助你自動分析專案並部署到 Netlify 雲端平台。

提供兩種使用方式：
  1. GUI 網頁介面 (推薦) - 視覺化操作，適合新手
  2. CLI 命令列介面 - 文字操作，適合進階使用者

================================================================================
使用方式一：GUI 網頁介面 (推薦)
================================================================================

  【啟動方式】

    方法 A：雙擊 run_gui.bat
    方法 B：python app.py

  【網址】

    http://127.0.0.1:5886

  【操作步驟】

    1. 輸入專案路徑 → 點擊「分析專案」
    2. 查看分析結果 → 確認配置選項
    3. 點擊「生成配置檔」
    4. 登入 Netlify → 初始化網站
    5. 點擊「預覽部署」測試
    6. 確認沒問題後點擊「正式部署」

================================================================================
使用方式二：CLI 命令列介面
================================================================================

  【啟動方式】

    方法 A：雙擊 run.bat
    方法 B：python netlify_ai.py

  【操作步驟】

    啟動後會顯示選單：

    [1] 指定專案路徑    - 輸入要部署的專案資料夾路徑
    [2] 使用當前目錄    - 部署目前所在的資料夾
    [3] 瀏覽選擇資料夾  - 輸入路徑後確認
    [0] 離開程式

    選擇後按照提示逐步操作即可。

================================================================================
使用前準備
================================================================================

  1. 安裝 Python 3.7+
     下載：https://www.python.org/downloads/

  2. 安裝 Flask (GUI 版本需要)
     pip install flask

  3. 安裝 Node.js（用於 Netlify CLI）
     下載：https://nodejs.org/

  4. 安裝 Netlify CLI
     npm install -g netlify-cli

  5. 註冊 Netlify 帳號
     https://app.netlify.com/

================================================================================
功能特色
================================================================================

  1. 自動分析專案
     - 偵測專案類型（靜態網站、Python Functions、Node.js）
     - 掃描檔案結構
     - 識別需要的環境變數

  2. 自動生成配置檔
     - netlify.toml（Netlify 部署配置）
     - .gitignore（排除敏感檔案）
     - .env.example（環境變數範例）
     - requirements.txt（Python 依賴）

  3. 對話式引導
     - 逐步確認配置選項
     - 可自訂發布目錄、Functions 目錄等

  4. 一鍵部署
     - 自動檢查 Netlify CLI
     - 引導登入流程
     - 執行預覽和正式部署

  • OPENAI_API_KEY     - OpenAI (GPT)
  • GOOGLE_API_KEY     - Google (Gemini)

================================================================================
常見問題
================================================================================

  Q: 出現「未偵測到 Netlify CLI」？
  A: 請執行 npm install -g netlify-cli 安裝

  Q: 登入失敗？
  A: 確認網路連線正常，或到 https://app.netlify.com/ 先登入一次

  Q: 部署失敗？
  A: 檢查 netlify.toml 設定是否正確，特別是：
     - functions 目錄路徑
     - requirements.txt 位置（需放在 functions 目錄內）

  Q: 環境變數沒有生效？
  A: 設定環境變數後需要重新部署才會生效

  Q: GUI 版本無法啟動？
  A: 請先安裝 Flask：pip install flask
